# Event Stormgng Part 3 - 可以做得更好

## 失敗的 Event Storming

- 當技術人員開始發瘋，開始狂聊技術細節如快取、資料庫等等。
- 當非技術人員開始發瘋，開始變成 Brain Storming。請記住， Event Storming 目的是讓團隊加速溝通流程而非發想新功能，那些請在事前的會議或使用者訪談中完成。
- 大家都坐下來時
- 只有一兩人在說話時

## 血和淚的教訓

- Domain Expert 必須要有自我意識
  - BDD Feature files
- Facilitator 必須獨立
